# V1.6 Deployment & Production Readiness Tasks

This checklist covers the transition from a stable local development build to a live production deployment.

## 1. 🏗️ Infrastructure Setup (Priority: High)
**Goal**: Establish the base infrastructure to host the AtlanticProxy backend and database.
- [ ] **Server Provisioning**:
    - [ ] Provision VPS (Ubuntu 22.04 LTS, 2GB+ RAM, 2 vCPU).
    - [ ] Configure SSH access and basic firewall (UFW: allow 22, 80, 443, 8082).
- [ ] **Environment Configuration**:
    - [ ] Create `production.env` with correct `OXYLABS_USERNAME`, `OXYLABS_PASSWORD`, and `PAYSTACK_SECRET_KEY` (Live).
    - [ ] Set `LOG_FORMAT=json` and `LOG_LEVEL=info`.

## 2. 🔐 Security & Networking (Priority: High)
**Goal**: Secure the application with SSL and proper isolation.
- [ ] **Reverse Proxy (Nginx)**:
    - [ ] Install Nginx.
    - [ ] Configure reverse proxy to `localhost:8082` (WebSocket support required).
- [ ] **SSL/TLS**:
    - [ ] Obtain Let's Encrypt certificate via Certbot for the API domain (e.g., `api.atlanticproxy.com`).
- [ ] **System Security**:
    - [ ] Create dedicated `atlantic` system user.
    - [ ] Configure Systemd service (`atlanticproxy.service`) with auto-restart and resource limits.

## 3. 💳 Live Billing Integration (Priority: Medium)
**Goal**: Switch from Test Mode to Live Mode for payments.
- [ ] **Stripe/Paystack Switch**:
    - [ ] Replace Test Public/Secret keys with Live keys in specific production config files (NOT in git).
    - [ ] Verify Webhook endpoint is accessible publicly.
- [ ] **Crypto Wallet**:
    - [ ] Verify static wallet addresses are correct for mainnet.

## 4. 🧪 Final Sanity Check (Priority: High)
**Goal**: Verify the deployed artifact works in the wild.
- [ ] **Connectivity Test**:
    - [ ] `curl -x https://api.atlanticproxy.com:8080` check.
- [ ] **Rotation Test**:
    - [ ] Verify IP rotation works with the Residential Proxy credentials on the live server.
- [ ] **Latency Check**:
    - [ ] Ensure round-trip time is acceptable (<500ms for proxy overhead).

---

### Definition of Done (Deployment)
1.  **Publicly Accessible**: API is reachable via HTTPS.
2.  **Live Payments**: Real money can be processed (or simulated safely in Live-Test).
3.  **Resilient**: Service auto-restarts on crash or reboot.
